Latest Pro Tip: use uv or pixi

Python environments, but not a lifestyle:

I have used pipenv, pyenv, poetry and conda/mamba. Current conclusion is simple: use uv for almost everything, keep pixi in reserve for ML stacks that really need Conda style system libraries.

Most of the pain is in three places:

  • project environments and Python versions
  • isolated CLI tools
  • AI and ML setups tied to CUDA, ROCm, BLAS and other binaries

uv covers the first two and is fine for most recent ML projects. When a project depends on a difficult CUDA or ROCm or Triton combination, I move that one project into pixi and freeze the environment there instead of touching the system. 

# new project
uv init smc-experiment
cd smc-experiment

# add deps
uv add numpy scipy librosa scikit-learn

# optional: pin Python
uv python install 3.11
uv python pin 3.11

# run code
uv run python scripts/whatever.py

# install a CLI tool
uv tool install ruff
ruff check .

Rule of thumb for future me: start with uv. If the install works and tests pass, stop thinking about it. If drivers or old binaries are a fight, that project gets a pixi environment and everything else stays simple.
