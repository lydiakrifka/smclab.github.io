Digital Signal Processing for Sound and Music

This page introduces the Digital Signal Processing (DSP) environment and practices used in the Sound and Music Computation master’s programme at Universitat Pompeu Fabra (UPF). It focuses on the MTG sms-tools and sms-tools-materials packages developed by Xavier Serra’s Music Technology Group for teaching sound analysis and synthesis.

Overview

sms-tools provides Python implementations of classic DSP models for sound analysis and synthesis, including:

  • dftModel.py — Discrete Fourier Transform models
  • stft.py — Short-Time Fourier Transform models
  • sineModel.py — Sinusoidal model
  • harmonicModel.py — Harmonic model
  • stochasticModel.py — Stochastic model
  • sprModel.py — Sinusoidal plus Residual model
  • spsModel.py — Sinusoidal plus Stochastic model
  • hprModel.py — Harmonic plus Residual model
  • hpsModel.py — Harmonic plus Stochastic model

Safe Installation and Environment Setup

Students are strongly encouraged to set up a self-contained environment using Python virtual environments. This avoids system-level dependency conflicts that can break Linux audio or multimedia packages. Always work in isolated project folders with venv or uv (recommended for reproducibility). 

        # Recommended setup (works on Linux, macOS, and Windows)
        # Install uv for fast, isolated dependency resolution:
        pip install uv

        # Create and activate a new environment
        uv venv .venv
        source .venv/bin/activate # (Mac/Linux)
        # .venv\Scripts\activate # (Windows PowerShell)

        # Install sms-tools and teaching materials
        pip install sms-tools
        git clone https://github.com/MTG/sms-tools-materials.git
        cd sms-tools-materials
        pip install -r requirements.txt

Why 100% Self-Contained?

On some Linux systems, a single broken library (e.g. libportaudiocpp or libfftw) can render sound drivers or Python audio backends unusable system-wide. By keeping your environment isolated, you ensure:

  • Reproducibility — everyone runs identical versions
  • Safety — your system audio and core packages remain untouched
  • Portability — you can zip your .venv and move it to another machine
  • Clean removal — delete the folder, and nothing breaks

Running the Material

Once dependencies are installed, explore the sms-tools-materials repository:

  • examples/ — Demonstrations of all models
  • exercises/ — Jupyter notebooks for class practice
  • interface-models/ — Graphical models GUI
  • interface-transformations/ — Transformations GUI
        # Run model interface
        cd interface-models
        python models_GUI.py

        # Run sound transformation GUI
        cd ../interface-transformations
        python transformations_GUI.py
