0. Dev Environment: uv & pixi

Most of the code on this blog is developed with a minimal tooling stack based on uv (from Astral) and, when needed, pixi. The idea is to avoid juggling multiple Python version / environment managers.

Development with Python

  • Use uv as the default for new and existing projects:
    • uv init — create a new project with a pyproject.toml.
    • uv pip ... — drop-in replacement for pip on legacy projects.
    • uv python install <version> — manage Python versions.
  • This replaces the usual stack of pipenv, pyenv, poetry, and ad-hoc virtualenvs. 
# new project
uv init smc-dsp-experiments
cd smc-dsp-experiments
uv add numpy scipy librosa

CLI tools written in Python

  • For standalone CLI tools, use uv tool instead of pipx:
    • uv tool install <package> — installs into an isolated env.
    • You can still pin / switch the Python version that tool uses.
# example: install a one-off CLI helper
uv tool install ruff
ruff check .

AI / ML projects

  • First try a plain uv environment; for many recent ML stacks (DSP notebooks, MIR baselines, smaller models) this is enough.
  • The main reason to reach for Conda-style environments is when you need specific BLAS / CUDA / system libraries or older combinations of packages.
  • If that happens, use pixi instead of a global Conda setup: you get per-project envs with the same “declare it, don’t manage it by hand” workflow.

For the SMC coursework shown here, the default assumption is: “just use uv, and only fall back to pixi if a particular ML stack really demands Conda-style pinning.”

3. Example Code Snippet

Minimal baseline for feature extraction and normalization (used in several experiments): 

# features.py – tiny baseline pipeline
import librosa, numpy as np
from sklearn.preprocessing import StandardScaler

def extract_log_mel(path, n_mels=64):
    y, sr = librosa.load(path, sr=None)
    mel = librosa.feature.melspectrogram(y=y, sr=sr, n_mels=n_mels)
    log_mel = librosa.power_to_db(mel, ref=np.max)
    scaler = StandardScaler()
    X = scaler.fit_transform(log_mel.T)  # frames × features
    return X, sr

X, sr = extract_log_mel("audio/example.wav")
print(X.shape, "sample rate:", sr)
